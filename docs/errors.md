# X.orb Error Code Reference

Every error response from the X.orb API follows a consistent JSON format:

```json
{
  "error": "Human-readable error message",
  "request_id": "req_1710720000000_abc123"
}
```

The `request_id` is included in every response and should be provided when contacting support for troubleshooting.

---

## Error Response Format

| Field | Type | Description |
|-------|------|-------------|
| `error` | string | A human-readable description of the error |
| `request_id` | string | Unique identifier for the request, useful for debugging |

---

## 400 Bad Request — Validation Error

The request body or query parameters failed validation.

**Common Causes:**
- Missing required fields (e.g., `name`, `role`, `sponsor_address`)
- Invalid field values (e.g., role not in allowed enum, name too short)
- Malformed JSON in request body
- Invalid action in PATCH request (must be `pause`, `resume`, or `renew`)

**Example Response:**

```json
{
  "error": "Validation failed: name must be at least 2 characters",
  "request_id": "req_1710720000001_def456"
}
```

**How to Fix:**
- Check the request body against the API documentation for the endpoint you are calling.
- Ensure all required fields are present and correctly typed.
- Agent names must be 2-64 characters. Descriptions must be under 256 characters.
- Valid agent roles are: `TRADER`, `RESEARCHER`, `MONITOR`, `CODER`, `GOVERNANCE_ASSISTANT`, `FILE_INDEXER`, `RISK_AUDITOR`.
- Batch requests accept 1-100 actions.

## 401 Unauthorized — Invalid API Key

Authentication failed. The API key is missing, malformed, invalid, revoked, or expired.

### Missing API Key

```json
{
  "error": "Missing x-api-key header",
  "request_id": "req_1710720000002_ghi789"
}
```

**How to Fix:** Include the `x-api-key` header in your request.

### Invalid Key Format

```json
{
  "error": "Invalid API key format. Keys must start with xorb_",
  "request_id": "req_1710720000003_jkl012"
}
```

**How to Fix:** Ensure your API key starts with `xorb_`. Keys generated by the API have the format `xorb_sk_<48 hex characters>`.

### Invalid or Revoked Key

```json
{
  "error": "Invalid API key",
  "request_id": "req_1710720000004_mno345"
}
```

**How to Fix:**
- Verify the API key is correct and has not been truncated.
- If you rotated your key, use the new key. The old key is revoked immediately upon rotation.
- Generate a new key via `POST /v1/auth/keys` if needed.

### Revoked Key

```json
{
  "error": "API key has been revoked",
  "request_id": "req_1710720000005_pqr678"
}
```

**How to Fix:** Generate a new API key via `POST /v1/auth/keys`.

## 402 Payment Required

The free tier has been exhausted and no valid x402 payment was provided, or the provided payment was rejected.

### Free Tier Exhausted

```json
{
  "error": "Payment Required",
  "status": 402,
  "details": {
    "endpoint": "POST /v1/actions/execute",
    "price_usdc": 0.005,
    "description": "Per-action gate check",
    "free_tier_exhausted": true,
    "free_tier_limit": 1000,
    "free_tier_used": 1000,
    "accepts": [
      {
        "network": "eip155:8453",
        "asset": "USDC",
        "amount": "5000"
      },
      {
        "network": "eip155:137",
        "asset": "USDC",
        "amount": "5000"
      }
    ],
    "payment_header": "x-payment",
    "payment_format": "base64(JSON({ signature, amount, network, nonce, expiry }))",
    "docs": "https://docs.xorb.xyz/payments"
  }
}
```

**How to Fix:**
- Wait for the free tier to reset at the beginning of the next calendar month.
- Include an `x-payment` header with a valid x402 payment. The `accepts` array tells you which networks and amounts are accepted.
- The amount is in USDC with 6 decimal places (e.g., `"5000"` = $0.005 USDC).

### Payment Rejected

```json
{
  "error": "Payment Rejected",
  "reason": "Insufficient payment: 3000 < 5000 (USDC, 6 decimals)",
  "status": 402
}
```

**How to Fix:**
- Ensure the payment amount meets or exceeds the required price.
- Verify the payment signature is valid and the nonce has not been reused.
- Check that the payment has not expired.

## 403 Forbidden

You are authenticated but not authorized to perform this action.

### Ownership Mismatch

```json
{
  "error": "Forbidden: you do not own this agent",
  "request_id": "req_1710720000006_stu901"
}
```

**How to Fix:** You can only manage agents that were registered with your wallet address as the `sponsor_address`. Verify you are using the correct API key (the one associated with the agent's sponsor address).

### Action Blocked by Pipeline

When an action is blocked by the 8-gate pipeline, the response has HTTP status 403:

```json
{
  "action_id": "act_1710720200000_def456",
  "agent_id": "agent_1710720000000_abc123",
  "approved": false,
  "gates": [
    { "gate": "identity", "passed": true, "latency_ms": 2 },
    { "gate": "role_auth", "passed": false, "reason": "Agent role RESEARCHER not authorized for tool 'transfer_funds'", "latency_ms": 1 }
  ],
  "reputation_delta": -2,
  "audit_hash": "sha256:d7a8fbb...",
  "timestamp": "2026-03-18T10:03:20.000Z",
  "latency_ms": 3
}
```

**How to Fix:**
- Check the `gates` array to find which gate blocked the action and why.
- Common blocks: role not authorized for the tool, reputation too low, rate limit exceeded within pipeline, risk score too high, compliance violation.
- Adjust the action, tool, or agent role as needed.

### Sponsor-Only Operations

```json
{
  "error": "Only the sponsor can renew this agent",
  "request_id": "req_1710720000007_vwx234"
}
```

**How to Fix:** Operations like renew, pause, resume, and revoke require the API key associated with the agent's sponsor address.

## 404 Not Found

The requested resource does not exist.

```json
{
  "error": "Agent not found",
  "request_id": "req_1710720000008_yza567"
}
```

**How to Fix:**
- Verify the agent ID, webhook ID, or listing ID in your request URL.
- The agent may have been revoked. Revoked agents may not be retrievable depending on the operation.
- Check for typos in the resource identifier.

---

## 429 Too Many Requests

You have exceeded the rate limit for your tier.

```json
{
  "error": "Rate limit exceeded. Try again in 3 seconds.",
  "request_id": "req_1710720000009_bcd890"
}
```

**Response Headers:**

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests allowed in the current window |
| `X-RateLimit-Remaining` | Requests remaining in the current window |
| `X-RateLimit-Reset` | Unix timestamp when the rate limit resets |
| `Retry-After` | Seconds to wait before retrying |

**How to Fix:**
- Wait for the duration specified in the `Retry-After` header before retrying.
- Implement exponential backoff in your client.
- If you consistently hit rate limits, consider upgrading your tier. See [Rate Limits](./rate-limits.md).
- Use batch endpoints (`POST /v1/actions/batch`) to reduce the number of individual requests.

---

## 500 Internal Server Error

An unexpected error occurred on the server.

```json
{
  "error": "Internal server error",
  "request_id": "req_1710720000010_efg123"
}
```

**How to Fix:**
- This is a server-side issue. Retry the request after a short delay.
- The official SDKs automatically retry 5xx errors up to 3 times with exponential backoff.
- If the error persists, contact support at contact@xorb.xyz with the `request_id`.
- Internal error details are never exposed to clients for security reasons. The `request_id` allows the X.orb team to investigate.

---

## 503 Service Unavailable

The service is temporarily unavailable, usually due to maintenance or a configuration issue.

```json
{
  "error": "Service configuration error. Contact support.",
  "request_id": "req_1710720000011_hij456"
}
```

**How to Fix:**
- This may indicate scheduled maintenance. Check the X.orb status page or `GET /v1/health`.
- Retry after a few minutes.
- If the issue persists, contact support at contact@xorb.xyz.
- For key rotation, this error indicates that the required database is temporarily unavailable.

## Error Handling Best Practices

### TypeScript SDK

The TypeScript SDK throws `XorbAPIError` for all non-2xx responses:

```typescript
import { XorbClient, XorbAPIError } from 'xorb-sdk'

const client = new XorbClient({
  apiUrl: 'https://api.xorb.xyz',
  apiKey: 'xorb_sk_...',
})

try {
  const result = await client.actions.execute({
    agent_id: 'agent_...',
    action: 'query',
    tool: 'web_search',
  })
} catch (err) {
  if (err instanceof XorbAPIError) {
    console.error(`API Error ${err.status}: ${err.message}`)
    console.error(`Request ID: ${err.requestId}`)

    switch (err.status) {
      case 401:
        // Re-authenticate or rotate key
        break
      case 402:
        // Handle payment required
        break
      case 429:
        // Wait and retry
        break
      case 500:
      case 503:
        // SDK handles retries automatically for 5xx
        break
    }
  }
}
```

### Python SDK

The Python SDK raises `XorbAPIError` for all non-2xx responses:

```python
from xorb import XorbClient, XorbAPIError

client = XorbClient(
    api_url="https://api.xorb.xyz",
    api_key="xorb_sk_...",
)

try:
    result = client.actions.execute(
        agent_id="agent_...",
        action="query",
        tool="web_search",
    )
except XorbAPIError as e:
    print(f"API Error {e.status}: {e}")
    print(f"Request ID: {e.request_id}")

    if e.status == 401:
        # Re-authenticate or rotate key
        pass
    elif e.status == 402:
        # Handle payment required
        pass
    elif e.status == 429:
        # Wait and retry
        pass
```

## Summary Table

| Status | Error Code | Description | Retryable |
|--------|-----------|-------------|-----------|
| 400 | `validation_error` | Request body or parameters failed validation | No (fix request) |
| 401 | `invalid_api_key` | API key is missing, invalid, or revoked | No (fix auth) |
| 401 | `expired_api_key` | API key has expired | No (generate new key) |
| 402 | `payment_required` | Free tier exhausted, payment needed | No (pay or wait for reset) |
| 403 | `forbidden` | Not authorized for this resource | No (use correct key) |
| 403 | `ownership_mismatch` | You do not own this agent | No (use correct key) |
| 404 | `not_found` | Resource does not exist | No (fix resource ID) |
| 429 | `rate_limit_exceeded` | Too many requests | Yes (wait for `Retry-After`) |
| 500 | `internal_error` | Unexpected server error | Yes (retry with backoff) |
| 503 | `service_unavailable` | Service temporarily down | Yes (retry after delay) |
